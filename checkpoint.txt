Data do Checkpoint: 28/05/2026

✅ 1. Frontend (React/Vite) & UX Híbrida
- UX Híbrida (Interface Inteligente): O sistema identifica se o usuário está no computador ou celular de forma nativa.
- Desktop (>768px): Cards com efeito de levitação 3D, rotação e brilho (Glow) azulado ao passar o mouse.
- Mobile (<768px): Efeito "Tactile Press" (card encolhe simulando um botão físico) e Glow intenso ao toque, eliminando falhas de rotação.
- Responsividade Avançada: Implementação de um "Grid Inteligente" usando CSS auto-fill e controle rígido das proporções das capas dos protocolos com aspect-ratio: 3/4, evitando distorções.
- Painel Admin: Sistema de busca filtrada em tempo real por categorias, envio em massa de PDFs, otimização das miniaturas e formulários alinhados.

🌐 2. Infraestrutura, API & Resolução de Nomes (DNS)
- API Auto-Detect: Lógica no apiService.js que detecta o IP do navegador automaticamente usando window.location.hostname. O sistema roda de forma simultânea em localhost:3001 e na rede do hospital sem configurações manuais no código.
- Novo Padrão de Hostname: Migração definitiva da identificação da máquina física de trabalho do Marcelo para o padrão corporativo atual (DIWGP-0004.saovicente.lan).
- Implantação de CNAME no Active Directory: Criação e homologação do registro do tipo Alias (CNAME) chamado "protocolos" diretamente no servidor de DNS central do hospital (IWGP-ADDS02), dentro da zona 'saovicente.lan'.
- Blindagem e Resiliência de Rede: O apelido foi mapeado para o hostname fixo da máquina, garantindo que mesmo que o roteador altere o IP dinâmico da máquina de trabalho do Marcelo (atualmente no IP 192.168.0.196), o acesso dos usuários não sofra interrupções.
- Link Oficial Homologado na Home (192.168.0.11): http://protocolos.saovicente.lan:3001/ (garantindo resolução estável em 100% das estações, tablets assistenciais e Wi-Fi corporativo).

⚙️ 3. Automação de Backend e Servidor
- Servidor Unificado Node.js: Arquitetura sem dependência do Apache (XAMPP). O Node.js/Express 5 atua servindo a API e o build do Frontend via rota curinga (/.*/).
- Processos em Segundo Plano: O sistema roda ininterruptamente como um serviço gerenciado em segundo plano pelo PM2 sob o nome "portal-protocolos".
- Scripts Facilitadores (Batch):
  * DEPLOY_AUTOMATICO.bat: Compila o Vite, limpa diretórios antigos, injeta a nova versão estável e reinicia o serviço no PM2 com 1 clique.
  * BACKUP_TOTAL_V3.bat: Rotina redundante que gera o dump do Banco de Dados (MySQL) e salva o espelhamento completo do projeto nas unidades C: (Local) e E: (Disco Externo BlueFrog).

📘 4. Documentação, Governança e Ecossistema (Google Workspace)
- Sincronização de Documentação: O arquivo README.md e o painel visual do sistema foram atualizados para refletir a nova versão 3.1.3.
- Alinhamento do Hub Central: Preparação da topologia de rede para integração transparente com o Hub Centralizador do hospital desenvolvido em Google Sites. O botão de "Protocolos Internos" passa a apontar para a URL amigável e qualificada, abstraindo a complexidade técnica para médicos e enfermeiros.
- Segurança de Registro: Configuração do CNAME com restrições de escrita no Active Directory, impedindo alterações não autorizadas por usuários comuns da rede corporativa (LGPD Compliance).
